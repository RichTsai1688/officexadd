# (一) 實施策略（請分年列述）

## 1. 規劃研究之產業領域、合作機構或社會應用議題概述（含營運現況與情境）、升級瓶頸與未來技術需求

本計畫聚焦於**醫療照護、長照與高風險場域的「智慧巡檢」升級**。典型營運情境包含病房/照護空間的例行巡檢（護理/照護人員定時巡房）、設備監測（監護儀、輸注設備）、環境安全（走道障礙物、地面濕滑），以及資產盤點（器材數量、位置與必要欄位）。目前多數場域已具備攝影機或影像資料來源，但巡檢主要仍依賴人力經驗與人工記錄，存在「人力不足、資訊延遲、跨系統難整合」等問題。

目前升級瓶頸可歸納為：

* **多任務、多情境：**巡檢任務不是單一模型可涵蓋，且病床、走道、設備區混合，導致需求複雜、介面不易收斂。
* **可用性與信任：**誤報（尤其跌倒、Alarm、濕滑）會造成現場信任快速崩壞，並可能引發告警疲乏。
* **影像品質變動：**遮擋、反光、角度偏移、夜間光源變化，使單點模型在真實場域不穩。
* **系統維運困難：**模組更新頻繁，若耦合過深，任何一處更新可能造成整體行為漂移。
* **資料治理與合規：**影像資料牽涉隱私、留存、調閱與權限控管，導入門檻高。

未來技術需求對應到本計畫的核心設計：

* **以 AI Agent 進行「場景路由」與「任務編排」**：在看到特定物件/場景時才啟動相對應任務，避免全跑造成資源失控。
* **導入 Skill（技能）層**：把每個工具調用變成可重用、可稽核的技能單元（含啟動條件、輸出規格、證據要求、降級策略）。
* **以 LoRA/QLoRA 進行參數效率調整**：不重新訓練基礎模型，而以低成本方式提升路由、技能選擇與結構化輸出一致性。
* **可靠度工程：品質分級＋證據優先＋去重節流**：把「不確定就降級」與「避免重複告警」做成系統規則，而不是靠單一分數門檻。

**分年策略**

* **第1年：**完成需求盤點與場域情境建模（病床/走道/設備區），建立端到端原型（觀測→路由→技能→事件）。
* **第2年：**在合作機構場域導入試點（pilot），完成八大模組「可插拔整合」與可靠度設計（品質分級、降級、去重）。
* **第3年：**擴大到多區域/多點位驗證，形成可移轉的部署與維運方法（KPI儀表板、回歸測試、版本治理），並推進產業化導入。

## 3. 針對產業及合作機構升級之特定挑戰／社會議題，提出具體解決方案規劃

本計畫的具體解決方案是一套「**智慧巡檢 AI Agent 平台**」，用**場景路由**把巡檢拆解成可控、可驗證、可擴充的工作流；並以**Skill 技能層**確保工具調用一致與可稽核；再以**LoRA/QLoRA**低成本修正路由與保守策略，使系統在真實場域逐步變可靠，而不需要重新訓練整個基礎模型。

**特定挑戰 → 對應解法**

* **挑戰A：多場景輸入＋多模組整合難**
  → **解法：場景路由（先判斷在床/走道/設備區）＋可插拔模組**，只啟動必要任務。
* **挑戰B：誤報造成信任崩壞（跌倒/Alarm/濕滑）**
  → **解法：品質分級＋證據優先＋保守降級輸出（unknown/需確認）＋去重節流**。
* **挑戰C：影像品質不穩（遮擋/反光/夜間）**
  → **解法：品質評分做 Gate**；品質不足時不做定量估測、不做高風險強結論。
* **挑戰D：版本更新導致行為漂移難維運**
  → **解法：決策層/執行層分離＋Skill 契約固定＋Golden set 回歸測試**，模組可換但行為可控。
* **挑戰E：資料與合規限制**
  → **解法：最小化資料暴露（只傳 ROI/特徵）、權限控管、稽核日誌、學生進場規範化**。

**分年策略**

* **第1年：**聚焦高風險任務（T1跌倒/姿勢、T2離床、T6儀器alarm）建立「可用且保守」的事件輸出。
* **第2年：**補齊八大任務（含T3/T4估測、T5時間窗生命徵象、T7環境風險、T8盤點），完成 pilot KPI。
* **第3年：**多區域擴散與維運制度化（回歸測試、模型/技能更新流程、場域回饋機制）。

## 4. 實施方法

### (1) 計畫整體研究規劃及預定最終目標（含導入場域應用規劃）

本計畫整體規劃採「先可用、再變強、再擴散」策略：

* **最終目標：**建置一套可在合作機構落地的「智慧巡檢 AI Agent 平台」，可輸入場域影像資料，自動完成**八大任務**之動態路由與執行，輸出可稽核的巡檢事件（含證據、品質、信心與降級原因），並能以版本治理方式長期維運。
* **導入場域規劃：**
  先選擇具代表性的單一/少數區域（如特定病房走道＋部分床位＋設備區）做 pilot；通過 KPI 後再擴大到多區域。
* **擴大社會應用意義：**
  除醫療照護外，架構可延伸至長照機構、公共安全、工安巡檢與智慧建築管理等多元場域，形成可擴散的「多工具巡檢 Agent」能力。

**分年策略**

* **第1年：**完成端到端閉環與高風險事件保守輸出，建立初版看板/報表。
* **第2年：**完成場域 pilot，八大任務可插拔整合，建立回歸測試與資料迭代流程。
* **第3年：**多區域擴散與制度化維運（模型/技能版本治理），形成技轉與擴散方案。

---

### (2) 計畫技術發展說明：技術（服務）功能與規格及年度里程碑

**核心功能（服務）**

* 場景辨識與物件偵測（前段輕量化）：床、人、點滴、尿袋、螢幕、地面、資產。
* **AI Agent 路由**：根據場景、物件、品質與狀態記憶，決定要跑哪些任務（只跑必要任務）。
* **Skill 技能層**：將每個任務工具調用規格化（啟動條件、輸入輸出 schema、證據、降級、去重）。
* **八大任務工具（T1–T8）**：可插拔替換、可獨立升級。
* 事件化輸出：每筆事件包含「類型/嚴重度/信心/品質/證據/原因」，支援看板、API 或資料庫串接。
* 版本治理：Golden set 回歸測試、KPI 儀表板與灰度上線流程。

**關鍵規格（建議寫法，以「目標」呈現，避免過度承諾）**

* **即時性：**高風險事件目標在「數秒級」產出可用事件（依場域網路與算力調整）。
* **可靠度：**對高風險事件優先壓低誤報；品質不足時明確降級（unknown/需確認）。
* **可維運：**任務模組可替換升級；路由與輸出格式穩定可回歸測試。
* **合規：**資料最小化、權限控管、稽核日誌。

**年度里程碑**

* **第1年（M1）：**

  * 完成架構骨架：前段偵測＋品質分級＋路由＋Skill 契約＋事件化輸出。
  * 完成高風險模組優先：T1/T2/T6 的場域可用版本（保守輸出）。
  * 建立 Golden set v1 與回歸測試流程。
* **第2年（M2）：**

  * 八大任務工具整合完成（T1–T8），技能庫與依賴關係固定化。
  * LoRA/QLoRA：完成路由與技能選擇的參數效率調整（不重訓基座），提升觸發精準率與結構化輸出穩定。
  * pilot 場域驗證（KPI 與使用者回饋）。
* **第3年（M3）：**

  * 多區域擴散驗證與維運制度化（版本治理、事件去重節流策略、告警可行動性）。
  * 完成技轉包：部署手冊、維運SOP、教育訓練教材與擴散方案。

### (4) 解決方案之場域驗證規劃（包含學生實作方案）

場域驗證採「由小到大」：

* **第1年：**控制環境＋小規模場域資料回放驗證（離線）、建立 KPI 計算與事件可回放機制。

* **第3年：**擴大至多區域或多點位，驗證版本更新不退步（回歸測試通過率）、維運流程可行、回饋機制有效。

**學生實作納入方式（場域驗證的一部分）**

* 建立「學生可負責、可交付」的子任務：資料治理、品質標註、KPI報表、技能契約測試、回歸集維護、場域SOP協作等，確保學生參與不影響場域安全與合規。

### (5) 學生於合作機構場域實作方案規劃與創新解決方案的關聯

本計畫的學生實作不是旁支，而是直接嵌入「Skill＋LoRA迭代」的核心流程：

* **對 Skill 的貢獻：**學生可依實際場域案例，協助整理「啟動條件、降級策略、證據需求」並做契約測試，形成技能庫的可維運資產。
* **對 LoRA/QLoRA 的貢獻：**學生可協助蒐集「低信心/矛盾/高風險」案例，並在專家指導下完成輕量標註（例如：該情境應輸出 unknown），形成高價值訓練樣本。
* **對八大任務的關聯：**每位學生可對應一至兩項任務模組（或一項任務＋其 Skill 驗證），讓學習與產出可衡量。

### (7) 預計可能遭遇之困難及解決途徑

* **困難1：影像品質不穩（遮擋/反光/夜間）**
  → **解法：品質分級 Gate＋保守降級輸出**；品質不足不做定量估測、不做高風險強結論。
* **困難2：誤報造成現場抗拒**
  → **解法：證據優先＋去重節流＋高風險事件先壓誤報**，以可行動性作驗收指標。
* **困難3：多模組整合與版本漂移**
  → **解法：決策/執行分離＋Skill 契約固定＋Golden set 回歸測試**；採灰度上線。
* **困難4：資料治理與合規限制**
  → **解法：最小化資料暴露（ROI/特徵）、權限控管、稽核日誌、學生進場規範化**。
* **困難5：場域需求變動或跨單位協調成本**
  → **解法：以 KPI 儀表板與例會節奏固化回饋，需求以技能契約增量方式納入**，避免大改。

**合作機構回饋機制**

* 灰度上線：新版本先在限定區域跑，通過 Golden set 回歸與KPI門檻才擴大。

**擴大社會應用**

* 以框架為核心，場域差異透過「新增/替換工具模組＋技能契約」適配：

  * 長照：離床/跌倒/呼吸等高風險優先
  * 公共安全：走道障礙/濕滑、群聚異常
  * 工安：危險姿勢、設備異常、區域風險
